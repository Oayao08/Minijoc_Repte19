Aquesta pàgina explica, de manera clara i ordenada, com està construït el joc: des de l’estructura HTML fins a la lògica JavaScript, passant per les fases, els obstacles i els missatges educatius.

1. Funcionament general del joc

El joc està pensat com una experiència interactiva en 2D dins d’un canvas. El jugador controla un patinet o vehicle similar i ha d’avançar per una carretera amb perspectiva, esquivant obstacles i respectant diferents normes de circulació.

Menú inicial El joc comença amb una pantalla principal on es mostren les instruccions bàsiques i el botó per començar.
Compte enrere Abans de començar, apareix un 3, 2, 1 i un missatge d’inici perquè el jugador es prepari.
Tutorial A la primera fase es practiquen els controls: accelerar, frenar i canviar de carril.
Fases progressives El joc afegeix dificultat a poc a poc: cotxes, semàfors, vianants, passos de zebra i distraccions digitals.

L’objectiu no és només puntuar, sinó aprendre a conduir amb seguretat i prendre decisions correctes en cada situació.

2. Estructura de l’HTML utilitzada

L’HTML està pensat per ser simple i net. La idea principal és tenir una pàgina amb un títol, diverses seccions explicatives i un espai per mostrar el codi de manera ordenada.

Exemple bàsic d’HTML comentat

<main class="contenidor">
        <header>
            <h1>Com ho hem fet el joc</h1>
            <p>Explicació clara del projecte.</p>
        </header>

        <section>
            <h2>Funcionament general</h2>
            <p>Aquí es descriu com va el joc.</p>
        </section>
        </main>

Aquesta estructura ajuda a separar el contingut en blocs fàcils de llegir i de revisar. A més, el títol principal i les seccions permeten entendre ràpidament de què tracta cada part.

3. Funcionament del codi JavaScript

El JavaScript és la part que dóna vida al joc. Gestiona l’estat del joc, els controls, les fases, els obstacles, els àudios i el dibuix constant de la pantalla.

Estat del joc El codi defineix si estem al menú, al tutorial, a una fase concreta o a la pantalla de final de partida.
Sistema de fases Cada fase introdueix un tipus de repte diferent i canvia els objectius del jugador.
Renderitzat A cada fotograma es dibuixen el fons, la carretera, els obstacles i la interfície.
Interacció Les tecles controlen el moviment, la velocitat i les decisions davant dels popups.

Exemple comentat de JavaScript: canvi de fases

// Si encara no estem a l'última fase...
function checkPhaseUp(dt) {
  if (phase >= PHASES.length - 1) return;

  // Sumem el temps passat
  phaseTimer += dt;

  // Quan la fase dura prou, pugem a la següent
  if (phaseTimer >= curPhase().duration) {
    phase++;
    phaseTimer = 0;
    phaseInTimer = 5000;   // Mostra la pantalla d'introducció
    st = GS.PHASE_IN;      // Canvia l'estat del joc
    applyPhase();          // Ajusta dificultat i obstacles
  }
}

Exemple comentat de JavaScript: entrada del jugador

document.addEventListener('keydown', e => {
  keys[e.key] = true;

  // Esquerra i dreta per canviar de carril
  if ((e.key === 'ArrowLeft' || e.key === 'a') && lCooldown <= 0) {
    if (pl.lane > -1) {
      pl.lane--;
      lCooldown = 210;
    }
  }

  if ((e.key === 'ArrowRight' || e.key === 'd') && lCooldown <= 0) {
    if (pl.lane < 1) {
      pl.lane++;
      lCooldown = 210;
    }
  }
});

4. Què fa cada part del joc?

Fons i perspectiva Es dibuixen el cel, els edificis i la carretera amb un efecte de profunditat per donar sensació de moviment.
Obstacles Apareixen cotxes, caixes, vianants i patinets. Cada element té una mida i un efecte visual diferent.
Popups educatius Quan el jugador accepta una distracció, surt un missatge explicatiu per reforçar l’aprenentatge.
Puntuació i salut El joc registra punts, distància recorreguda, encerts i errors del jugador.

També hi ha efectes d’àudio, canvis de carril suaus, moviments animats i una pantalla de final de partida amb resum del rendiment.

5. Resum de l’estructura del joc

  • Menu: pantalla d’inici amb instruccions.
  • CD: compte enrere abans de començar.
  • TUTO: tutorial per aprendre els controls.
  • PLAY: fase principal amb obstacles i puntuació.
  • PHASE_IN: pantalla d’introducció entre fases.
  • POPUP / ERROR: missatges especials per distraccions o errors.
  • GO: game over i resum final.

Aquesta organització fa que el joc sigui més fàcil de mantenir, entendre i ampliar amb noves idees.

6. Conclusió

En aquest projecte, l’HTML serveix per presentar la informació de manera clara, mentre que el JavaScript és el motor que fa funcionar tota la interacció del joc. Gràcies a l’estructura per fases, el jugador aprèn progressivament i entén millor les normes bàsiques de seguretat vial.

Si es vol ampliar el projecte, es poden afegir més nivells, més obstacles, nous sons, més missatges educatius o una puntuació més detallada.
